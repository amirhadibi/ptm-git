PTM Front-end

The purpose of PTM is to match tabular financial data that comes from Praedea to existing accounts in Wells. PTM first matches the accounts programmatically based on the literal. Then the user matches the remaining unmatched accounts or adds new accounts. There are also various functions to allow the user to manipulate the data to conform to Wells standards.

Documents:

Shows a list of documents with their status in a grid. In this page, you can sort and search data by clicking on column headers.

Details:

You can select a document and then see the details of that document by selecting Details in the menu bar. Details page shows a list of accounts for the selected document with their amounts in a grid.
